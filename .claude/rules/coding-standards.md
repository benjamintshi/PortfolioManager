# 编码规范

## 不可变性
- 不修改原对象，返回新对象
- 使用 `{ ...obj }` 或 `[...arr]` 做浅拷贝
- `const` 优先，`let` 仅在必须重新赋值时

## 错误处理
- 不吞掉异常（禁止空 `catch`）
- 记录日志：`console.error('context:', error)` 含上下文
- UI 层给用户友好提示，服务端记录详细日志
- 使用 try/catch 包裹所有异步操作

## 输入校验
- 在系统边界校验（API路由、表单、文件读取）
- SQL 用 `?` 占位符，绝不拼接字符串
- 不信任外部数据（API响应、用户输入）

## 文件组织
- 单文件 200–400 行，上限 800 行
- 高内聚、低耦合
- 业务逻辑放 `services/`，不放在路由中
- 一个文件一个职责

## 命名
- 文件/变量: camelCase
- 类/接口/类型: PascalCase
- 常量: UPPER_SNAKE_CASE
- 路由文件名与资源对应: `assets.ts`, `prices.ts`
