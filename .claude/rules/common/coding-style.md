# 编码风格 (通用)

> Language note: 以下规则可被语言特化规则覆盖。

## 代码质量
- 函数只做一件事，< 50 行
- 单文件 200–400 行，上限 800 行
- 禁止魔法数字 — 用命名常量
- 禁止死代码 — 删除，不要注释掉
- 禁止 TODO/FIXME 进入提交 — 必须关联 issue
- DRY: 复制粘贴 ≥3 次就提取

## 不可变性
- 默认不可变：不修改原对象，返回新对象
- 优先使用 const / final / let（不可变绑定）

## 命名
- 名字要能描述意图，不要缩写
- 布尔变量以 is/has/should/can 开头
- 函数以动词开头: `calculateTotal`, `findById`

## 文件组织
- 高内聚低耦合
- 按功能/领域组织，不按类型
- 公共接口在文件顶部
