# 错误处理 (通用)

## 原则
- 永远不要静默吞掉异常
- 错误信息必须包含上下文（什么操作失败、输入是什么）
- 使用类型化错误 / 自定义错误类，不用通用 Error
- 日志分级: error=故障, warn=可恢复, info=事件, debug=细节

## 边界处理
- 所有外部输入必须校验（API参数、用户输入、文件内容）
- 所有外部调用必须设超时
- 外部 API 失败必须有降级方案或重试
- 数据库操作必须在事务中（涉及多步写入时）

## 禁止
- 空 catch 块
- 用默认值掩盖错误（如 `unwrap_or(0)`, `|| ''`）
- 在日志中打印 secrets/tokens/密码
