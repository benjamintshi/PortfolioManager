# 性能 (通用)

## 原则
- Profile before optimize — 不要猜瓶颈
- 大数据集用懒加载和分页
- 缓存昂贵计算（带 TTL + 大小上限）
- 避免 N+1 查询 — 批量处理

## 数据库
- WHERE / JOIN / ORDER BY 列加索引
- 查询必须有 LIMIT — 禁止无界查询
- 连接池 — 不要每请求创建连接
- 批量 INSERT/UPDATE 用于大批数据

## 内存
- 循环内不做内存分配 — 提前创建复用
- 所有 Map/Cache 设大小上限 + 清理
- 定时器/监听器用完必须清理 — 防泄漏

## I/O
- 所有网络请求设超时
- 能异步/后台处理的不要同步阻塞
- 响应压缩 (gzip/brotli)
