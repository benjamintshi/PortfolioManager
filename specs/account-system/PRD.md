# Portfolio Manager 2.0 — 全面重设计 PRD

## 概述
以「账户体系」为基础，重新设计整个投资组合管理系统的信息架构和页面结构。

## 设计原则
1. **账户为根** — 所有资产挂在 平台 → 子账户 → 持仓 层级下
2. **一屏掌控** — Dashboard 一眼看到总资产、盈亏、平台分布、配置偏离
3. **数据自动化** — 交易所 API 同步为主，手动录入为辅
4. **决策导向** — 每个页面都应引导到"下一步该做什么"

## 信息架构（6 个板块）

```
💼 PORTFOLIO
├── 📊 总览 Dashboard        — 全局数据一屏掌控
├── 🏦 账户 Accounts         — 平台/子账户/持仓管理 + API 同步
├── 📦 持仓 Holdings         — 跨平台合并持仓视图（只读）
├── ⚖️ 配置 Allocation       — 目标配比 + 再平衡建议（合并旧 Rebalance）
├── 📈 分析 Analytics        — 风险/相关性/历史表现（合并旧 Analysis）
└── 🗓️ 市场 Market           — 宏观指标 + 事件日历（合并旧 MacroEvents + MacroIndicators）
```

### 移除/合并的旧页面
| 旧页面 | 去向 |
|--------|------|
| Assets | → Holdings（只读视图）+ Accounts（编辑操作）|
| Rebalance | → Allocation（扩展） |
| Analysis | → Analytics（更名） |
| Roadmap | → Allocation 的子 tab「执行计划」 |
| InvestmentPlans | → Allocation 的子 tab「建仓计划」 |
| MacroEvents | → Market |
| MacroIndicators (组件) | → Market + Dashboard 概览 |

## 各页面详细设计

### 1. 📊 Dashboard（总览）
**目标**：30 秒内掌握组合全貌

**区块**：
1. **Hero** — 总资产（大数字）+ 日/周/月涨跌幅 + USD/CNY 切换
2. **Stat Cards (4 列)**
   - 总资产值
   - 总盈亏（金额 + 百分比）
   - 今日变动
   - 最大偏离度（点击跳转 Allocation）
3. **双饼图** — 左: 按资产类别分布 | 右: 按平台分布
4. **净值曲线** — 历史趋势（7d/30d/90d/1y 切换）
5. **宏观指标横条** — Fear & Greed / VIX / DXY / BTC Dom（精简版，点击跳转 Market）
6. **平台概览卡片** — 每个平台一张小卡：名称、总值、同步状态

### 2. 🏦 Accounts（账户管理）
**目标**：管理所有平台、配置 API、同步数据、手动录入

**区块**：
1. **Hero** — 标题 + 「添加平台」「记录转账」按钮
2. **Stat Cards** — 平台数 / 总资产 / 已同步 / 最后同步时间
3. **Tab**: 全部平台 | 转账记录 | 同步日志
4. **平台卡片列表**（每个平台一张 glass card）：
   - 头部：平台图标 + 名称 + 同步状态 badge + 总资产 + 盈亏 + 同步/编辑按钮
   - 展开：子账户列表 → 每个子账户下的持仓表格
   - 手动平台：每行有编辑/删除按钮 + 底部「添加持仓」
   - exchange 平台：持仓只读（API 同步），但可配置 API Key
5. **弹窗**：
   - API Key 配置（密码框 + 安全提示）
   - 添加平台
   - 记录转账（从 → 到 → 资产 → 数量 → 手续费）
   - 手动添加/编辑持仓

### 3. 📦 Holdings（持仓一览）
**目标**：跨平台聚合视图，按资产维度查看

**区块**：
1. **Hero** — 标题 + USD/CNY 切换
2. **筛选栏** — 搜索框 + 类别下拉 + 平台下拉
3. **合并持仓表格**：
   - 同一 symbol 跨平台合并显示（如 BTC 在币安+Bybit 合并为一行）
   - 列：资产名/代码 | 总数量 | 均价 | 现价 | 总市值 | 盈亏 | 分布平台
   - 展开行：显示各平台明细（哪个子账户持有多少）
   - 纯只读，不可编辑（编辑去 Accounts）
4. **底部汇总** — 总资产数 / 总市值

### 4. ⚖️ Allocation（配置管理）
**目标**：合并再平衡 + 路线图 + 投资计划

**区块**：
1. **Hero** — 标题 + 「配置目标」按钮
2. **Tab**: 当前配置 | 建仓计划 | 执行路线图

**Tab 1: 当前配置**
- 目标配比 stat cards（每个类别一张）
- 状态横条（需要调整 / 配置均衡）+ 最大偏离度
- 调仓建议列表（带进度条可视化）

**Tab 2: 建仓计划**（原 InvestmentPlans）
- 计划列表 + 分批执行状态
- 添加/编辑计划

**Tab 3: 执行路线图**（原 Roadmap）
- 按阶段分组的计划执行进度
- 分析报告列表

### 5. 📈 Analytics（分析中心）
**目标**：风险评估 + 历史表现

**区块**：
1. **Hero** — 标题 + 「重新分析」按钮
2. **Stat Cards** — 年化收益 / 年化波动率 / 夏普比率 / 最大回撤
3. **相关性矩阵**（3 列卡片）
4. **最优配比建议**
5. **分析说明**

### 6. 🗓️ Market（市场动态）
**目标**：合并宏观指标 + 事件日历

**区块**：
1. **Hero** — 标题 + 「添加事件」「录入指标」按钮
2. **Tab**: 宏观指标 | 事件日历

**Tab 1: 宏观指标**
- 指标卡片网格（Fear & Greed / VIX / DXY / US 10Y / BTC Dom / USDCNY）
- 每张卡片：当前值 + 颜色编码 + 来源 + 更新时间
- 可添加自定义指标

**Tab 2: 事件日历**
- 即将到来 / 全部 / 已过期 tab
- 事件列表（日期 + 名称 + 类型 badge + 重要性 badge）
- 展开显示影响资产 / 预期影响 / 实际结果

## 验收标准

### 账户体系
- AC-1: 可创建平台 + 子账户，删除级联
- AC-2: Binance API Key 配置后同步现货+理财持仓
- AC-3: Bybit API Key 配置后同步统一账户持仓
- AC-4: 支付宝手动添加/编辑/删除持仓
- AC-5: 跨平台转账记录，双端持仓正确变更
- AC-6: API Key 加密存储，响应脱敏

### 页面
- AC-7: Dashboard 双饼图（类别+平台）正确显示
- AC-8: Holdings 跨平台合并持仓，展开显示平台明细
- AC-9: Allocation 三个 tab 数据正确
- AC-10: Market 指标自动更新 + 事件日历
- AC-11: 导航栏 6 个入口，路由全部可达

### 数据
- AC-12: 旧 assets 数据清空，API 同步 + 手动录入重建
- AC-13: Portfolio summary/history/risk API 读取 holdings 表

## 范围外
- 自动交易/下单
- 交易历史同步（只同步当前持仓快照）
- 支付宝 API 接入
- 合约保证金/杠杆详细计算
- 投资计划关联账户（后续迭代）
